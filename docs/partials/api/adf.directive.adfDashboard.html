adfDashboard
directive in module adf

Description

adfDashboard is a directive which renders the dashboard with all its components. The directive requires a name attribute. The name of the dashboard can be used to store the model.

Usage

as element:
<adf-dashboard
       name="{string}"
       [editable="{boolean}"]
       [collapsible="{boolean}"]
       [maximizable="{boolean}"]
       [enable-confirm-delete="{boolean}"]
       [structure="{string}"]
       [adf-model="{object}"]
       [adf-widget-filter="{function}"]
       [continuous-edit-mode="{boolean}"]
       [categories="{boolean}"]>
</adf-dashboard>
as attribute
<div adf-dashboard
     name="{string}"
     [editable="{boolean}"]
     [collapsible="{boolean}"]
     [maximizable="{boolean}"]
     [enable-confirm-delete="{boolean}"]
     [structure="{string}"]
     [adf-model="{object}"]
     [adf-widget-filter="{function}"]
     [continuous-edit-mode="{boolean}"]
     [categories="{boolean}"]>
   ...
</div>

Directive info

  • This directive creates new scope.

Parameters

ParamTypeDetails
namestring

name of the dashboard. This attribute is required.

editable
(optional)
boolean

false to disable the editmode of the dashboard.

collapsible
(optional)
boolean

true to make widgets collapsible on the dashboard.

maximizable
(optional)
boolean

true to add a button for open widgets in a large modal panel.

enableConfirmDelete
(optional)
boolean

true to ask before remove an widget from the dashboard.

structure
(optional)
string

the default structure of the dashboard.

adfModel
(optional)
object

model object of the dashboard.

adfWidgetFilter
(optional)
function

function to filter widgets on the add dialog.

continuousEditMode
(optional)
boolean

enable continuous edit mode, to fire add/change/remove events during edit mode not reset it if edit mode is exited.

categories
(optional)
boolean

enable categories for the add widget dialog.
